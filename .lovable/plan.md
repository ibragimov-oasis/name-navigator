# План: довести раздел «Для людей» до полезного состояния

Раздел `/people` уже существует (профили, взрослые, новообращённые, персонаж, псевдоним, история, совместимость), но половина инструментов работает «один раз и забыл». Этот план закрывает реальные дыры — **то, без чего раздел остаётся витриной, а не помощником**.

## Что добавляем (по приоритету)

### 1. Активный профиль реально проникает в инструменты
Сейчас `?name=...` подставляется только при клике из `PersonCard`. Если человек активировал профиль и пошёл в `/tafsir`, `/numerology`, `/dna`, `/signature`, `/certificate` — они его не видят.

- В каждой из этих страниц: если URL не содержит `?name=`, а есть `activePerson` — подставить его автоматически и показать плашку «Контекст: <имя> · сменить».
- `/certificate` — авто-подстановка `name`, `person` (отношение), `date` (рождение) из активного профиля.
- `/compare` — кнопка «Сравнить с активным профилем».
- Глобальный `ActivePersonBanner` (тонкая полоса под `Header`) — виден на всех инструментах, кликабелен → `/people/profiles`.

### 2. Редактирование, заметки и история имени
Профиль сейчас можно создать и удалить — больше ничего.

- `PersonForm` переиспользуем в режиме «Редактировать» (передаём `initial`, меняем `submitLabel`).
- В `Person` добавить `nameHistory: { name: string; reason?: string; date: string }[]` — фиксирует прежние имена (особенно важно для новообращённых и при смене имени).
- На `PersonCard` — кнопка «Редактировать» рядом со звездой и таймлайн смен имён.
- Расширенные заметки: `tags` (массив строк), `meaningPersonal` (что имя значит лично для пользователя).

### 3. Полный генератор насаба / куньи / арабского ФИО
Сейчас есть `/wizard`, но он не интегрирован с профилем. Нужно:

- Новая страница `/people/nasab` — собирает арабское ФИО из активного или выбранного профиля: «Кунья (Абу/Умм X) · Имя · ибн/бинт Отец · Семья · ал-Нисба · ал-Лакаб».
- Авто-генерация куньи из имени старшего ребёнка (если есть профиль с `relation: "child"` и связь `parentId`).
- Связь профилей: добавить `parentId?: string` и `spouseId?: string` в `Person` — это сразу даёт «семейное древо имён».
- Кнопки: скопировать арабское написание (через `arabicTranslit.ts`), скопировать кириллицу, открыть сертификат.

### 4. Углубить flow для новообращённого
Сейчас `RevertName` подбирает 5 имён по смыслу — но имена берутся из общего детского каталога. Нужно адаптировать под взрослого мусульманина.

- Шаг «Сохранить имя» в конце wizard'а: автоматически создать `Person` с `relation: "self"`, выбранным именем, исходным именем в `nameHistory`.
- Шаг «Шахада-чек-лист»: имя выбрано → шахада произнесена → гусль сделан → имам подтвердил (галочки сохраняются в профиле).
- Для каждого из 5 имён добавить:
  - Арабское написание (`arabicTranslit.ts`).
  - Произношение (TTS на `ar-SA` через `tts.ts`).
  - Кнопка «Сделать моим именем» (создаёт `Person` с этим именем).
- Дополнить `revertGuides` в `peopleKnowledge.ts` хадисами о смене имени и списком «имён, которые менял Пророк ﷺ».

### 5. Семейное древо и совместимость по ролям
- Раздел `/people/profiles`: компактная визуализация связей — родители → дети → супруги (на основе `parentId`/`spouseId`).
- В `Compatibility` — авто-подбор: «Проверить совместимость со всеми членами семьи» (один клик → список со всеми сохранёнными `Person`).
- Расширить `personCompatibility.ts`:
  - Особая логика для пары `parent ↔ child` (важно созвучие имени и отчества).
  - Особая логика для `spouse ↔ spouse` (дополнительный балл за совместимость происхождения / культуры).
  - Учитывать `nameHistory` — если имя сменено, считать совместимость по новому имени, но в пояснении упомянуть прежнее.

### 6. Углубить «Имя для персонажа» и «Псевдоним»
- `CharacterName`: после генерации — кнопка «Сохранить как профиль персонажа» (`relation: "character"`).
- `Pseudonym`: тоже сохраняется как `relation: "character"` с заметкой «псевдоним».
- В `Pseudonym` добавить: проверка `.com/.io`-доступности (опционально, через простой fetch к whois API; если падает — мягко скрываем).
- Для персонажа: история генераций сохраняется в `localStorage` (`imyagen.character.history.v1`), лента из 10 последних.

### 7. Расширить базу исторических личностей и связать с именами
Сейчас 12 персон — мало для «справочника». Добавить минимум 20 новых:

- Женщины (Хадиджа, Айша, Фатима аз-Захра, Зейнаб бинт Али, Рабия аль-Адавия — суфийская поэтесса).
- Полководцы (Халид ибн аль-Валид, Тарик ибн Зияд, Нур ад-Дин Занги).
- Учёные (Аль-Газали, Ибн Хальдун, Ибн Баттута).
- Нерелигиозные значимые (Низами, Хайям — уже отчасти есть).

В каждой записи добавить поле `derivedNames?: string[]` — какие современные имена восходят к этой личности. Это даёт обратную ссылку: открыл имя «Халид» в каталоге → видит, в честь кого названо.

### 8. Карточки и карта раздела
- `People.tsx` — добавить вверху блок «Активный профиль» с фото-аватаром (буква), быстрыми действиями (Тафсир / Нумерология / Сертификат) — чтобы хаб был осмыслен и для постоянного пользователя.
- Хлебные крошки на всех `/people/*`-страницах.
- На главной (`Index.tsx`) — если есть активный профиль, заменить «Имя дня» на «Имя для <имя профиля>» (персональная рекомендация).

## Технические детали (для разработчика)

- **Миграция** `localStorage` (`imyagen.people.v1`): при загрузке если у `Person` нет `nameHistory` — добавить пустой массив; ничего не удаляем. После миграции писать ключ `imyagen.people.v2`.
- **Связи**: `parentId`/`spouseId` хранят `Person.id`. При удалении профиля — пройтись по остальным и обнулить ссылки (без cascade delete).
- **Активный профиль в инструментах**: в каждой странице с `useSearchParams` добавить:
  ```ts
  const { activePerson } = usePeople();
  const nameFromUrl = searchParams.get("name");
  const effectiveName = nameFromUrl || activePerson?.fullName || "";
  ```
  Если используется `activePerson` (а не URL) — показать плашку «Контекст: <имя>» с кнопкой «сменить» (открывает `/people/profiles`).
- **TTS для арабского** (`speakName(name, "ar-SA")`): хелпер `src/lib/tts.ts` уже есть, нужно убедиться что подбирает арабский голос; если в системе нет — fallback на `ru-RU` транслит.
- **Арабское написание имён**: `src/lib/arabicTranslit.ts` уже есть, переиспользуем для `Nasab.tsx` и `RevertName.tsx`.
- **`derivedNames` на исторических фигурах**: добавить опциональное поле; в `HistoricalFigures.tsx` показывать как чипы с ссылкой `/?name=...`. Обратная ссылка: в `NameCard` (где есть `namedAfter`) — связать с записями из `historicalFigures.ts` по совпадению строки.
- **Семейное древо**: чистый CSS/SVG (без новых зависимостей). Простая иерархическая раскладка на основе `parentId`.
- **Никаких новых пакетов**. Всё делается на существующих `lucide-react`, `react-router-dom`, `@tanstack/react-query`.

## Порядок реализации

1. Миграция схемы `Person` + интеграция активного профиля во все инструменты (даёт 80% ценности сразу).
2. Редактирование, история имени, заметки, связи (`parentId`/`spouseId`).
3. `/people/nasab` и `ActivePersonBanner`.
4. Углубление revert-flow (арабский, TTS, чек-лист, сохранение).
5. Расширение исторических личностей (+20) и связки с именами.
6. Доработка совместимости (роли, семья) и сохранения в `CharacterName` / `Pseudonym`.
7. Персонализация главной и хаба `/people`.

Каждый пункт самодостаточен — можно остановиться после любого, и сайт будет лучше, чем сейчас.

## Что НЕ делаем

- Авторизация и синк через Supabase — ждём, когда инфраструктура понадобится массово.
- PWA / мобильное приложение.
- Платная подписка.
- Замена существующих страниц `/children`, `/wizard`, `/pets` — только дополняем.
- Новые AI-вызовы там, где можно обойтись локальной логикой.
