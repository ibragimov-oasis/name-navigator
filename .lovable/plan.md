

## Аудит и прокачка раздела «Подпись» (NameSignature)

### Текущее состояние

**Что есть:** Страница `/signature` с поиском имени, 8 стилей подписей, генерация через Canvas 2D, скачивание PNG.

**Главная проблема:** `LETTER_PATHS` — пустой объект `{}`. Все буквы рендерятся через `defaultPath()`, которая генерирует почти одинаковые загогулины по `charCode`. Результат — все буквы выглядят одинаково, подпись нечитаема и некрасива (видно на скриншоте: повторяющиеся одинаковые петли).

**Чего не хватает:**
- Реальные пути для кириллических и латинских букв
- Разнообразие форм между стилями (сейчас разница только в цвете/наклоне)
- Обработка загрузки, анимация генерации
- Возможность выбрать избранную подпись
- Кнопки быстрого доступа к популярным именам
- Закрытие dropdown при клике вне него

---

### Баги и проблемы

1. **`LETTER_PATHS` пуст** — ни одна буква не определена, `defaultPath` генерирует одинаковые формы для всех символов
2. **Нет `useEffect`** для закрытия dropdown — выпадающий список подсказок не закрывается при клике вне
3. **`RotateCcw` импортирован, но не используется** — мёртвый импорт
4. **`useEffect` импортирован, но не используется**
5. **Тройной вызов `ctx` в строке 281** — `connPoints.length > 1 ? ctx : ctx` — бессмысленный тернарник
6. **Нет обработки ошибок** — если canvas context null, тихий сбой
7. **Отсутствует мемоизация** `allNames` — вызов `getChildNames()` каждый рендер

---

### План реализации (5 этапов)

#### Этап 1: Капитальный ремонт движка (`signatureGenerator.ts`)

Полная переработка системы рисования букв:

- **Определить `LETTER_PATHS` для 33 кириллических + 26 латинских букв** — каждая буква как массив контрольных точек, моделирующих реальное рукописное написание (не просто петли, а формы букв: «М» с двумя горбами, «А» с шапкой, «Р» с петлёй)
- **Добавить вариативность форм по стилю:**
  - Business: компактные, аккуратные буквы с лёгким наклоном
  - Presidential: крупная первая буква, остальные мельче, размашистый росчерк
  - Calligraphic: тонкие-толстые переходы, высокие буквы
  - Quick: только 2-3 буквы читаемы + длинный хвост
  - Royal: золотой цвет, декоративные завитки на первой букве
  - Arabic: имитация арабского написания справа налево с точками
  - Minimalist: только инициалы, одним движением
  - Artistic: экспрессивные петли, драматичные перепады давления
- **Улучшить давление пера:** переход от 3 фиксированных проходов к динамическому расчёту ширины по кривой давления (толще в начале буквы, тоньше к концу)
- **Добавить параметр `letterSpacing`** в стиль для контроля плотности
- **Улучшить flourish:** разные типы росчерков (петля, волна, спираль) вместо одного синусоидального

#### Этап 2: Улучшение UI страницы (`NameSignature.tsx`)

- **Быстрые кнопки** — 6-8 популярных мусульманских имён (Мухаммад, Али, Фатима, Аиша, Юсуф, Ибрахим) для мгновенной генерации
- **Skeleton-загрузка** — показывать скелетоны карточек пока canvas рендерит
- **Анимация появления** — карточки плавно появляются с задержкой (staggered)
- **Закрытие dropdown** при клике вне компонента (useEffect + ref)
- **Кнопка «Обновить»** (RotateCcw) — перегенерировать все подписи с другим seed
- **Добавить в избранное** — отмечать лучшие подписи звёздочкой, сохранять в localStorage
- **Выбор фона** — белый/кремовый/прозрачный для каждого стиля
- **Поделиться** — кнопка копирования подписи в буфер обмена как изображение
- **Адаптив** — на мобильном (612px viewport) сетка 1 колонка, крупные кнопки скачивания

#### Этап 3: Фикс всех багов

- Убрать неиспользуемые импорты (`useEffect` если не нужен, `RotateCcw` — задействовать)
- Убрать бессмысленный тернарник `connPoints.length > 1 ? ctx : ctx`
- Мемоизировать `allNames` через `useMemo`
- Добавить null-check для canvas context
- Типизировать всё строго (убрать неявные `any`)

#### Этап 4: Связи с другими разделами

- **Из карточки имени** (`NameCard.tsx`) — кнопка «Создать подпись» → ссылка на `/signature?name=Имя`
- **Из NameDNA** — секция «Подпись для этого имени» с preview
- **Из NameTafsir** — показать подпись имени арабским стилем
- **URL-параметр** — поддержка `?name=X` для прямого перехода с именем

#### Этап 5: Новые фичи

- **Сравнение подписей** — выбрать 2 стиля и показать рядом в увеличенном виде
- **Анимация рисования** — кнопка «Показать как рисуется» — подпись рисуется по точкам с анимацией (requestAnimationFrame)
- **SVG-экспорт** — помимо PNG, генерация SVG для масштабирования без потери качества
- **Подпись на документе** — preview подписи на шаблоне документа/письма (мокап)
- **История генераций** — последние 10 сгенерированных подписей в localStorage

### Технические детали

**Пример реалистичного пути для буквы «М»:**
```text
stroke 1 (левая ножка):    [0,30] → [2,15] → [3,0]
stroke 2 (первый горб):    [3,0] → [8,20] → [12,0]  
stroke 3 (второй горб):    [12,0] → [16,20] → [20,0]
stroke 4 (правая ножка):   [20,0] → [21,15] → [22,30]
```

**Динамическое давление вместо фиксированных 3 проходов:**
```text
for each segment i of N:
  t = i / N
  pressure = bell_curve(t) × style.pressure
  lineWidth = base × pressure × (1 + jitter)
  → толсто в середине, тонко на краях
```
